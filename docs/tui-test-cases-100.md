# TUI 입력 테스트 케이스 100개

평가자는 이 테스트 케이스를 사용하여 TUI를 직접 실행하고 결과를 기록하세요.

테스트 방법:
1. `./bin/kosis` 실행
2. 각 케이스의 "입력" 순서대로 키 입력
3. "기대 결과"와 실제 결과를 비교
4. Pass/Fail 기록

## 평가자 사용법

1. `./bin/kosis` 실행
2. 위 표의 "입력" 열 순서대로 키 입력
3. "기대 결과"와 실제 동작 비교
4. Pass/Fail 기록
5. Fail인 경우 "비고" 열에 실제 동작 기록
6. 평가 완료 후 `docs/tui-eval-reviewer*.md`에 점수 반영

### python3 pty 자동 테스트

직접 실행이 어려운 항목은 아래 패턴으로 자동 테스트:

```python
import subprocess, time, os, pty, select, struct, fcntl, termios

master, slave = pty.openpty()
winsize = struct.pack('HHHH', 50, 120, 0, 0)
fcntl.ioctl(master, termios.TIOCSWINSZ, winsize)
proc = subprocess.Popen(['./bin/kosis'], stdin=slave, stdout=slave, stderr=slave, close_fds=True)
os.close(slave)

def drain(t=3):
    buf = b''
    end = time.time() + t
    while time.time() < end:
        r, _, _ = select.select([master], [], [], 0.2)
        if r:
            try: buf += os.read(master, 16384)
            except: break
    return buf.decode('utf-8', errors='replace')

# 초기 대기
time.sleep(2); drain(2)

# 테스트 입력
os.write(master, '인구'.encode('utf-8'))
time.sleep(0.5)
os.write(master, b'\r')
time.sleep(8)

# 결과 확인
import signal
os.kill(proc.pid, signal.SIGWINCH)
time.sleep(2)
screen = drain(3)
# screen 내용 분석...

os.write(master, b'q')
time.sleep(1)
proc.wait()
```
