# 평가자2 체크리스트

담당:
- `cmd/quick.go`
- `internal/nlp/matcher.go`
- `internal/interactive/interactive.go`

평가 원칙:
- quick은 설계서의 핵심 편의기능이므로 실제 실행까지 확인한다.
- 대화형 UX는 문서 설명과 사용 흐름이 맞아야 한다.
- 100점 미만이면 바로 재작업 요청한다.
- 문서/도움말/실행 결과가 서로 다르면 실행 결과를 우선하고 불일치로 감점한다.

평가 사전 문서:
- `../../docs/superpowers/specs/2026-03-31-kosis-cli-design.md`
- `../../skills/kosis-cli/SKILL.md`

## 점수 정책 (총 100점)
- A. quick 규칙 기반: 40점
- B. quick AI 경로: 30점
- C. matcher 정확도: 20점
- D. interactive UX: 10점
- 100점 기준: 모든 치명 항목 통과 + 총점 100점
- 치명 항목 1개라도 실패 시 해당 라운드는 최대 79점

치명 항목:
- [ ] 규칙 기반 quick이 "명령 제안"이 아니라 실제 조회 실행까지 완료된다.
- [ ] AI 경로가 생성 명령을 실제 실행한다.
- [ ] quick 완료 후 재사용 가능한 원라인 명령이 출력된다.
- [ ] 설계서 대표 예시(서울 미분양 최근 6개월) 재현 가능

## A. quick 규칙 기반
- [ ] `kosis q "서울 미분양 최근 6개월"`이 조회 완료까지 이어진다. (10점)
- [ ] `kosis q "GDP 2020~2024"`가 기간을 반영해 실행된다. (8점)
- [ ] `kosis q "인구 2015,2020,2025"`가 비연속 시점을 반영한다. (8점)
- [ ] quick 완료 후 재사용 원라인 명령이 출력된다. (8점)
- [ ] 오류 발생 시 원인 + 다음 단계(예: meta/data 직접 실행)가 안내된다. (6점)

## B. quick AI 경로
- [ ] `--ai <tool>` 경로에서 생성된 명령이 실제 실행된다. (12점)
- [ ] 실행 전 확인 흐름(또는 명시적 스킵 옵션)이 일관된다. (6점)
- [ ] `-f/--format`, `-o/--output`이 생성 명령 실행 시 보존된다. (6점)
- [ ] 실패 시 AI 도구 미설정/실행 실패 에러가 구체적으로 안내된다. (6점)

## C. matcher
- [ ] 지역(전국/서울 등), 기간(최근 N, 범위), 비연속 시점 파싱이 동작한다. (8점)
- [ ] 주기(월/연/분기/반기) 추론 또는 명시 처리 결과가 일관된다. (4점)
- [ ] 바로가기 사전이 SKILL 주요 예시(GDP/인구/미분양/물가)를 커버한다. (4점)
- [ ] 파싱 실패 시 크래시 없이 안전하게 fallback 한다. (4점)

## D. interactive UX
- [ ] 인자 없는 quick/data에서 대화형 흐름이 정상 진입/종료된다. (4점)
- [ ] 프롬프트 문구가 입력 기대값을 명확히 안내한다. (3점)
- [ ] 기본값 및 취소/빈입력 처리 정책이 일관된다. (3점)

## 재현 명령 기록
- [ ] `go run . quick --help`
- [ ] `go run . quick "서울 미분양 최근 6개월"`

- [ ] `go run . quick "인구 2015,2020,2025"`
- [ ] `go run . quick "서울 미분양 최근 6개월" --ai claude`
