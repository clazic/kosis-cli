# 15. AI 통계 조회 표준 워크플로우

AI가 자연어 요청을 kosis-cli 명령어로 변환할 때의 표준 절차입니다.

## 단계별 흐름

```
사용자 요청 (자연어)
  ↓
1단계: 검색 (search) → 통계표 찾기
  ↓
2단계: 메타 확인 (meta) → 구조 파악
  ↓
3단계: 가용 시점 확인 → 데이터 존재 여부
  ↓
4단계: 데이터 조회 (data) → 정확한 파라미터로
  ↓
5단계: 결과 출력 → -f md로 Markdown 테이블 (데이터 축소 금지)
```

---

## 1단계: 검색 — 통계표 찾기

**원칙**: 키워드는 짧고 핵심적으로. 연도/숫자 코드 사용 금지.

```bash
# ✅ 좋은 예
kosis s "미분양" -n 10
kosis s "제조업 주요지표" -n 10
kosis s "인구총조사" -n 10

# ❌ 나쁜 예
kosis s "2023년 울산 제조업 생산액"     # 연도 포함 → 에러
kosis s "11501"                         # 숫자 코드 → 에러
kosis s "광업 제조업 산업중분류 시도별 주요지표 2023"  # 너무 긴 키워드
```

**검색 결과에서 확인할 것:**
- `ORG_ID` + `TBL_ID` → 2단계에서 사용
- `TBL_NM` → 원하는 통계표인지 확인
- `STRT_PRD_DE` / `END_PRD_DE` → 수록 기간 대략 파악

---

## 2단계: 메타 확인 — 구조 파악

```bash
kosis m <ORG_ID> <TBL_ID>
```

**메타에서 확인할 3가지:**

### (1) 분류 (`-c1`, `-c2`, ...)

```
[분류] objL 파라미터에 사용
  objL → 분류ID: A  분류명: 시도별     ← -c1에 해당
    00               전국
    11               서울
    26               울산

  objL → 분류ID: B  분류명: 산업별     ← -c2에 해당
    0                광업 및 제조업
    C10              식료품 제조업
    C24              1차 금속 제조업
```

- **분류 순서**: 메타에 나열된 순서대로 `-c1`, `-c2`, `-c3`, ...
- **코드 사용**: 한글 이름이 아닌 왼쪽의 코드를 사용
- **레벨 구조**: 코드 길이로 레벨 구분

### (2) 항목 (`-i`)

```
[항목] itmId 파라미터에 사용
    T01              사업체수
    T07              생산액
    T08              부가가치
```

### (3) 수록주기 (`-p`)와 수록기간

```
[수록정보] prdSe 파라미터에 사용
    prdSe=년          ← -p Y
    prdSe=5년         ← -p F (주의!)
```

---

## 분류 코드 레벨 구조

KOSIS 분류 코드는 길이로 레벨을 구분합니다:

```
코드        레벨      예시
─────────────────────────────────────────
0           총계      광업 및 제조업
B           대분류    광업(05~08)
B05         중분류    석탄 원유 및 천연가스 광업
B051        소분류    석탄 광업
B0510       세분류    석탄 광업
B05100      세세분류  석탄 광업

C           대분류    제조업(10~34)
C10         중분류    식료품 제조업
C101        소분류    도축 육류 가공 및 저장 처리업
C1011       세분류    도축업
C10111      세세분류  육류 도축업(가금류 제외)
```

**코드 선택 규칙:**
- 전체 합계 → `0` 또는 `00` (총계 코드)
- 대분류 비교 → 대분류 코드 나열: `"C10+C20+C24"`
- 특정 항목의 하위 전체 → 와일드카드: `"C10*"`
- 전체 → `ALL`

---

## 3단계: 가용 시점 확인

**반드시 데이터 조회 전에 확인.** 일부 통계표는 특정 연도에 데이터가 없습니다.

```bash
# 최근 5개 시점 조회 → 가용 시점 파악
kosis d <ORG_ID> <TBL_ID> -c1 <코드> -c2 <코드> -i <항목> -p <주기> -l 5
```

**주의할 패턴:**
- 경제총조사 연도(2010, 2015, 2020)에 데이터가 빠진 통계표가 있음
- `prdSe=5년`인 통계표는 5년 단위로만 데이터 존재

---

## 4단계: 데이터 조회

```bash
kosis d <ORG_ID> <TBL_ID> \
  -c1 <분류1코드> \
  -c2 <분류2코드> \
  -i <항목코드> \
  -p <수록주기> \
  -s <시작시점> -e <종료시점>
```

---

## 파라미터 결정 체크리스트

| 파라미터 | 어디서 확인 | 주의사항 |
|----------|------------|----------|
| `-c1`~`-c8` | meta 분류 (순서대로) | 코드 사용, 한글 금지 |
| `-i` | meta 항목 | `T01`, `T07` 등 코드 |
| `-p` | meta 수록정보 (`prdSe=`) | `년`→Y, `5년`→F, `월`→M |
| `-s`/`-e` | 3단계 가용 시점 확인 후 | 없는 시점 → 에러 30 |
| `-l` | 최근 N개 | 가용 시점 확인용으로 먼저 사용 |

## 실전 예시: "울산 제조업 생산액 알려줘"

```bash
# 1. 검색
kosis s "제조업 주요지표" -n 5
# → ORG_ID=101, TBL_ID=DT_1FS1004

# 2. 메타 확인
kosis m 101 DT_1FS1004
# → c1: 시도별 (26=울산)
# → c2: 산업별 (0=광업및제조업, C10=식료품, ...)
# → 항목: T07=생산액
# → prdSe=년

# 3. 가용 시점 확인
kosis d 101 DT_1FS1004 -c1 26 -c2 0 -i T07 -p Y -l 3
# → 2017, 2018, 2019 (2015 없음 주의)

# 4. 데이터 조회
kosis d 101 DT_1FS1004 -c1 26 -c2 0 -i T07 -p Y -s 2017 -e 2019
```

## 실전 예시: "울산 식료품 제조업 소분류별 사업체수"

```bash
# 1~2. 위와 동일 통계표

# 3. 식료품(C10) 하위 소분류 코드 확인
kosis m 101 DT_1FS1004 | grep "C10"
# → C101 도축, C102 수산물, C103 과실채소, ...

# 4. 소분류 데이터 조회
kosis d 101 DT_1FS1004 -c1 26 -c2 "C101+C102+C103+C104+C105+C106+C107+C108" -i T01 -p Y -l 1
```

## 연관 통계표 찾기

하나의 통계 주제가 여러 통계표로 나뉘어 있는 경우가 많습니다. 연관 통계표를 찾는 3가지 방법:

### 방법 1: 키워드 변형 검색

같은 주제를 다른 키워드로 검색하면 관련 통계표를 더 찾을 수 있습니다.

```bash
# "산업분류별 주요지표" → 연도별로 다른 통계표 존재
kosis s "시도 산업분류별 주요지표" -n 10
# → DT_1FS1101 (2023~2024)  ← 최신
# → DT_1FS1001_S (2020~2022)
# → DT_1FS1001 (2007~2020)  ← 과거
# → DT_1FS1004 (2007~2019)  ← 다른 기준
```

**패턴**: TBL_ID 뒤에 `_S`가 붙은 것은 과거 버전, 번호가 다른 것은 다른 기준(종사자규모별, 출하액규모별 등)

### 방법 2: 계층 탐색 (`kosis ls`)

KOSIS는 통계표를 주제별 트리 구조로 분류합니다.

```bash
# 1. 최상위 주제 목록
kosis ls
# → A_4: 인구총조사, A_7: 주민등록인구, ...

# 2. 주제 하위 탐색
kosis ls --parent A_4
# → A11: 인구부문, A12: 가구부문, ...

# 3. 세부 목록 (실제 통계표)
kosis ls --parent A11
# → A11_2015_1: 총조사인구(2015년 이후)
# → A11_2010:   총조사인구(2010년)
# → A111:       총조사인구(1925~2010년)
```

**핵심**: 연도별로 분리된 통계표를 찾을 때 유용. 같은 조사가 시기별로 다른 통계표ID를 가짐.

### 방법 3: TBL_ID 패턴으로 유추

KOSIS 통계표 ID는 패턴이 있습니다:

| 패턴 | 의미 | 예시 |
|------|------|------|
| `DT_1FS1001` | 원본 | 시도/산업분류별 주요지표 (2007~2020) |
| `DT_1FS1001_S` | 후속 버전 | 시도/산업분류별 주요지표 (2020~2022) |
| `DT_1FS1101` | 개편 버전 | 시도/산업분류별 주요지표 (2023~) |
| `DT_1FS1002` | 같은 조사 다른 기준 | 종사자 규모별 |
| `DT_1FS1003` | 같은 조사 다른 기준 | 출하액 규모별 |

**활용**: 원하는 시점의 데이터가 없으면, TBL_ID의 숫자를 변형하여 검색하면 과거/최신 버전을 찾을 수 있음.

### 연도가 겹치지 않는 통계표 이어붙이기

```bash
# 2007~2020 데이터
kosis d 101 DT_1FS1001 -c1 26 -c2 0 -i T07 -p Y -s 2017 -e 2019

# 2020~2022 데이터 (후속 통계표)
kosis d 101 DT_1FS1001_S -c1 26 -c2 0 -i T07 -p Y -s 2020 -e 2022

# 2023~ 데이터 (개편 통계표)
kosis d 101 DT_1FS1101 -c1 26 -c2 0 -i T07 -p Y -s 2023 -e 2024
```

> ⚠ 주의: 통계표가 바뀌면 분류 코드도 달라질 수 있음. 각 통계표의 `meta`를 개별 확인 필요.

## 5단계: 결과 출력 규칙

### 출력 형식
- **사용자에게 보여줄 때**: `-f md` (Markdown 테이블) 권장
- **파이프/후처리용**: `-f json`
- **파일 저장**: `-o 파일명.xlsx` 등

```bash
# AI가 사용자에게 보여줄 때
kosis d 101 DT_1IN1502 -c1 "00+11+26" -i T100 -p Y -l 3 -f md
```

### 데이터 무단 축소 금지 (필수)

| 금지 행위 | 올바른 방법 |
|----------|------------|
| AI가 `--fields`를 임의 추가하여 컬럼 제한 | `--fields` 없이 전체 컬럼 출력 |
| 결과 행을 임의로 잘라서 일부만 표시 | 전체 행 표시. 많으면 "N건 조회됨" 안내 |
| "주요 항목만 정리했습니다" 식의 요약 | 원본 데이터 그대로 전달 후 설명 추가 |
| 분류/항목 코드를 AI 판단으로 축소 | 사용자가 지정한 범위 그대로 사용 |

**예외**: 사용자가 명시적으로 "주요 항목만", "요약해줘", "컬럼 줄여줘" 등을 요청한 경우만 허용.

## 에러 대응

| 에러 | 원인 | 해결 |
|------|------|------|
| 에러 30 + 검색 | 연도/숫자 코드 포함 | 키워드만 사용 |
| 에러 30 + 데이터 | 해당 시점에 데이터 없음 | `-l 5`로 가용 시점 확인 |
| 에러 21 | 잘못된 파라미터 | `meta`로 코드 재확인. 한글 이름 사용 금지 |
| 에러 31 | 4만 셀 초과 | 범위 축소 또는 `ALL` 대신 특정 코드 사용 |
