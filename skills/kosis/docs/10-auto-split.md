# 10. 자동 분할 조회 (4만 셀 초과)

`ALL`로 전체 분류/항목을 선택하면 4만 셀을 초과할 수 있습니다. 이 경우 자동으로 시점 축 분할 조회를 수행합니다.

```bash
kosis d 101 DT_1IN1502 -c1 ALL -i ALL -p Y -s 2015 -e 2024 -o 인구.db
# ⚠ 예상 셀 수: ~796,800 (자동 분할 조회)
# [1/20] 2015년 조회 중...
# ...
# ✓ 인구.db 저장 완료
```

## 동작 방식

1. Meta API로 예상 셀 수 계산 (분류 × 항목 × 시점 × 14)
2. 4만 이하 → 일반 조회
3. 4만 초과 → 시점 축으로 청크 분할 (Y/M/Q/H 단위)
4. API 키 1개 → 순차 실행, 2개 이상 → 워커 풀 병렬 실행

## 빈 구간 처리

데이터가 없는 시점 구간(API 오류 30)은 자동으로 건너뜁니다. 통계표 개편 등으로 특정 연도에 데이터가 없는 정상 상황입니다.

## 병렬 조회 속도 비교

| API 키 수 | 방식 | 100만 셀 예상 시간 |
|----------|------|------------------|
| 1개 | 순차 실행 | ~25분 |
| 2개 | 2-워커 병렬 | ~13분 |
| 3개 | 3-워커 병렬 | ~9분 |

```bash
# API 키 추가로 속도 향상
kosis config add-key <KEY2>
kosis config add-key <KEY3>
kosis config key-list     # 등록된 키 확인
```

## 자동 분할 비활성화

```bash
# 직접 범위를 관리하고 싶을 때
kosis d 101 DT_1IN1502 -c1 "00" -i ALL -p Y -s 2020 -e 2024 --no-auto-split
```
